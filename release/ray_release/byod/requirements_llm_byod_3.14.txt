# Pin the torch family to the build that the ray-llm base layer (and vLLM
# 0.22.0's prebuilt wheels) are compiled against. sentence-transformers pulls
# torch transitively, and this depset is otherwise unconstrained, so without
# these pins it resolves to the latest PyPI torch, overwriting the base layer's
# 2.11.0+cu130 and breaking vLLM/torchvision prebuilt extensions (undefined
# libtorch symbol _RegisterOrVerify / missing torchvision::nms op).
# torch 2.11.0+cu130 also brings the CUDA-13 runtime (libcudart.so.13) that vLLM
# 0.22.0's _C extension dlopens, so the cu13 nvidia-* packages no longer need to
# be pinned by hand (they did when torch was cu129 and only shipped the cu12
# runtime). Keep in sync with the rayllm deplock + llm-requirements.
